# Atom Components - Supabase Design System

## Overview

"Atom components are the fundamental building blocks of our design system. These are primitive, single-purpose components that serve as the foundation for building more complex interfaces."

These components form the smallest reusable units within the Supabase Design System architecture, distinguishing them from more complex fragment components and UI pattern guidelines.

## Foundation & Technology

The atom components are constructed using **shadcn/ui** as their base framework. They are housed within the `ui` package, enabling consistent reusability across all Supabase applications.

## Component Categories

The design system includes the following atom components:

**Form & Input Elements:** Accordion, Alert, Aspect Ratio, Avatar, Badge, Breadcrumb, Button, Calendar, Card, Carousel, Checkbox, Collapsible, Combobox, Command, Command Menu (cmdk), Context Menu, Date Picker, Dialog, Drawer, Dropdown Menu, Expanding Textarea, Field, Form, Input, Input OTP, Label, Textarea

**Interactive & Display:** Hover Card, Keyboard Shortcut, Menubar, Mermaid, Nav Menu, Navigation Menu, Pagination, Popover, Progress, Radio Group, Radio Group Card, Radio Group Stacked, Resizable, Scroll Area, Select, Separator, Sheet, Sidebar, Skeleton, Slider, Sonner, Switch, Table, Tabs, Toggle, Toggle Group, Tooltip, Tree View

**Dialog & Alerts:** Alert Dialog

## Key Characteristics

- **Base Layer Architecture:** Form the foundational layer ensuring visual and functional consistency throughout the design system
- **Composability:** Designed to combine together for creating fragment components and implementing UI patterns
- **Accessibility & Consistency:** Provide standardized styling, accessibility features, and predictable behavior across common interface elements
