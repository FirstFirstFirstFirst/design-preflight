# UI Patterns - Supabase Design System

## Overview

The UI patterns section provides "higher-level guidance on how to compose components together to solve common interface problems." These patterns differ from atomic components by offering blueprints for entire interface sections rather than individual building blocks.

## Purpose

UI patterns establish consistent approaches for addressing recurring design challenges across Supabase products, ensuring standardized solutions for similar features throughout the application.

## Pattern Categories

The design system documents six primary pattern types:

1. **Charts** - Standardized approaches to visualizing data with consistent styling
2. **Empty States** - Communicating data absence and guiding user actions
3. **Forms** - Creating cohesive experiences in page layouts and side panels
4. **Layout** - Establishing consistent page structures with proper spacing and content organization
5. **Markdown** - Rendering composable markdown with established defaults
6. **Navigation** - Organizing hierarchical navigation systems across products

## Technical Approach

UI patterns may incorporate external libraries (react-markdown, reactflow, recharts) or compose components from the ui package. They serve as "blueprints for solving recurring design problems."

## Context

The design system draws inspiration from Radix, shadcn/ui, and Geist, with source code available on GitHub.
